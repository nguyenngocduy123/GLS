How to update order?

1. Log into the system

API Documentation: Log in

Request

User role must be a planner

    // POST http://localhost/api/auth/v2/login
    {
        "username": "user_doe",
        "password": "123"
    }

Response

Subsequent requests MUST provide token in the request header[x-access-token]

    // HTTP/1.1 200 OK
    {
        "_id": "58118ee072e0c18ab626501e",
        "username": "user_doe",
        "fullname": "Planner Doe",
        "role": "planner"
        "usergroup": null,
        "forceChangePassword": true,
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9"
    }

2a. Update order details

API Documentation: Update Order

Request

Use this API only if you wish to modify the following fields:

  • Priority
  • VehicleRestriction
  • Customer contact details (i.e. the entity that initiated the order)
    • CustomerName
    • CustomerPhone
    • CustomerEmail
    // header['x-access-token'] = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9'
    // PUT http://localhost/web/data/v2.1/order/sg-12345
    {
        "newValues": {
            "Priority": 5
        }
    }

Response

    // HTTP/1.1 200 OK
    1 // one record was updated

2b. Update job details

API Documentation: Update Job

Request

Use this API only if you wish to modify the following fields:

  • JobType
  • NoteFromPlanner
  • Location (i.e. where to pick-up or deliver)
    • Address
    • Postal
    • Lat
    • Lng
  • Time (i.e. when to fulfill the job)
    • StartTimeWindow
    • EndTimeWindow
    • ServiceTime
  • Person-in-charge contact details (i.e. the person at the pick-up or delivery location)
    • ContactName
    • ContactPhone
    • ContactEmail
    // header['x-access-token'] = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9'
    // PUT http://localhost/web/data/v2.1/job/100
    {
        "newValues": {
            "ContactName": "Jane Doe",
            "ContactPhone": 98765432,
            "ContactEmail": null
        }
    }

Response

    // HTTP/1.1 200 OK
    1 // one record was updated

Request
