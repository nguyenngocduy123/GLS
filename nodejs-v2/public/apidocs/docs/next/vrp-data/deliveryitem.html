POST /web/data/v2.1/job/:jobId/item

Create job item(s) for one job. If any job item fails to create, the entire list of job items will not be created

The following columns cannot be overwritten with this API:

  • ActualItemQty
    • Default: null

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Params

FieldRequiredTypeDefault ValueDescription
jobIdrequiredIntegerId column of DeliveryDetail table

Request Body

FieldRequiredTypeDefault ValueDescription
recordrequiredObject[]List of job items to insert into the database
record[].ItemIdrequiredString(...30)Id of the item to pick-up or deliver
record[].ItemQtyrequiredFloatNumber of items to pick-up or deliver

Request Example

    // create two job items for job 100
    // POST http://localhost/web/data/v2.1/job/100/item
    {
        "record": [{
            "ItemId": "Bag",
            "ItemQty": 3
        }, {
            "ItemId": "Pallet",
            "ItemQty": 1
        }]
    }

Success Response

HTTP Status 200

FieldRequiredTypeDefault ValueDescription
requiredString[]List of Id of inserted records

Success Response Example

    // HTTP/1.1 200 OK
    [300, 301] // two records inserted

PUT /web/data/v2.1/jobItem/:jobItemId

Update job item details. Job item is identified based on jobItemId

Resource Info

Information
Response formatText
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Params

FieldRequiredTypeDefault ValueDescription
jobItemIdrequiredIntegerId column of DeliveryItem table

Request Body

FieldRequiredTypeDefault ValueDescription
newValuesrequiredObjectValues to overwrite existing record with
newValues.ItemIdoptionalStringId of the item to pick-up or deliver
newValues.ItemQtyoptionalFloatNumber of items to pick-up or deliver
newValues.ActualItemQtyoptionalFloatNumber of items that have been picked up and/or delivered

Request Example

    // http://localhost/web/data/v2.1/jobItem/100
    {
        "newValues": {
            "ItemQty": 15
        }
    }

Success Response

HTTP Status 200

FieldTypeDefault ValueDescription
IntegerNumber of updated job items

Success Response Example

    // HTTP/1.1 200 OK
    1 // one record was updated

GET /web/data/v2.1/jobItem

Get list of job items based on either, where filter or date filter

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Query String

One of the following filters must be present when querying. (i.e. not allowed to query all records from database)

FieldRequiredTypeDefault ValueDescription
whereoptionalStringWhere conditions in JSON string

Request Example
