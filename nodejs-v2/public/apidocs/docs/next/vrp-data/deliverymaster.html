POST /web/data/v2.1/order

Create order(s). If any order fails to create, the entire list of orders will not be created

The following columns cannot be overwritten with this API:

  • VehicleId
    • Default: null
  • LastAttemptedByDriver
    • Default: null
  • LastAttemptedByPlateNumber
    • Default: null

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Body

FieldRequiredTypeDefault ValueDescription
recordrequiredObject[]List of orders to insert into the database
record[].IdrequiredString(...30)Order identifier. Cannot contain symbols other than dashes, underscores and spaces
record[].PriorityoptionalInteger(1-10)2Ranking of jobs. Higher number has higher priority
record[].CustomerNameoptionalString(...254)Name of person who has placed this order
record[].CustomerPhoneoptionalString(8)Contact number of the person who has placed this order. Numbers only
record[].CustomerEmailoptionalString(...254)Email address of the person who has placed this order
record[].VehicleRestrictionoptionalStringComma-delimited string of vehicle restrictions for this order
record[].UserGroupoptionalString(...100)Usergroup assigned with this order
record[].DeliveryDetailsrequiredDeliveryDetail[]Jobs for this order
record[].DeliveryDetails[].DeliveryItemsoptionalDeliveryItem[]List of items to pick-up or delivery for each job
record[].DeliveryDetails[].VerificationCodeoptionalVerificationCodeVerification code for each job

Request Example

    // http://localhost/web/data/v2.1/order
    {
        "record": [{
            "Id": "sg-12345",
            "Priority": 1,
            "CustomerName": "ABC Company Pte. Ltd.",
            "CustomerPhone": 66123456,
            "CustomerEmail": "example@abc.com",
            "VehicleRestriction": null,
            "DeliveryDetails": [{
                "JobSequence": 1,
                "JobType": "PICKUP",
                "Address": "11 Tuas Link 1",
                "Postal": "639588",
                "Lat": 1.3341,
                "Lng": 103.636,
                "StartTimeWindow": "2018-01-01T09:00:00+08:00",
                "EndTimeWindow": "2018-01-01T13:00:00+08:00",
                "ServiceTime": 15,
                "NoteFromPlanner": [
                    { key: "Equipments", value: "Crane" }
                ],
                "ContactName": "John Doe",
                "ContactPhone": 91234567,
                "ContactEmail": null,
                "DeliveryItems": [
                    { "ItemId": "Bag", "ItemQty": 5 }
                ],
                "VerificationCode": {
                    "Code": "12345"
                }
            }]
        }]
    }

Success Response

HTTP Status 200

FieldRequiredTypeDefault ValueDescription
requiredString[]List of Id of inserted records

Success Response Example

    // HTTP/1.1 200 OK
    ["sg-12345"] // one record inserted

PUT /web/data/v2.1/order/:orderId

Update order details. Order is identified based on orderId

The following columns cannot be overwritten with this API:

  • VehicleId
    • Default: null

Resource Info

Information
Response formatText
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Params

FieldRequiredTypeDefault ValueDescription
orderIdrequiredStringId column of DeliveryMaster table

Request Body

FieldRequiredTypeDefault ValueDescription
newValuesrequiredObject[]Values to overwrite existing record with
newValues.LastAttemptedByDriveroptionalString(...100)Full name or username of the driver who has last attempted the order
newValues.LastAttemptedByPlateNumberoptionalString(...10)Plate number of the vehicle assigned to the driver who has last attempted the order
newValues.PriorityoptionalInteger(1-10)Ranking of jobs. Higher number has higher priority
newValues.CustomerNameoptionalString(...254)Name of person who has placed this order
newValues.CustomerPhoneoptionalString(8)Contact number of the person who has placed this order. Numbers only
newValues.CustomerEmailoptionalString(...254)Email address of the person who has placed this order
newValues.VehicleRestrictionoptionalStringComma-delimited string of vehicle restrictions for this order
newValues.UserGroupoptionalString(...100)Usergroup assigned with this order

Request Example

    // http://localhost/web/data/v2.1/order/sg-12345
    {
        "newValues": {
            "Priority": 5
        }
    }

Success Response

HTTP Status 200

FieldTypeDefault ValueDescription
IntegerNumber of updated orders

Success Response Example

    // HTTP/1.1 200 OK
    1 // one record was updated

GET /web/data/v2.1/order

Get list of orders based on either, where filter or date filter

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Query String

One of the following filters must be present when querying. (i.e. not allowed to query all records from database)

FieldRequiredTypeDefault ValueDescription
whereoptionalStringWhere conditions in JSON string
where.DeliveryDetailoptionalStringWhere conditions for jobs in JSON string
startDateoptionalStringGet jobs within a date range (startDate, endDate). Value must be in ISO8601 format
endDateoptionalStringGet jobs within a date range (startDate, endDate). Value must be in ISO8601 format
dateoptionalStringGet jobs on a date. Value must be in ISO8601 format

Request Example

  • Get orders assigned to vehicle Truck3Ft
    • http://localhost/web/data/v2.1/order?where={"VehicleId":""VehicleId": "Truck3Ft""}
  • Get orders with jobs at a location
    • http://localhost/web/data/v2.1/order?where={"DeliveryDetail":{"Postal":"639588"}}
  • Get orders between 1 January 2018 and 31 Feburary 2018
    • http://localhost/web/data/v2.1/order?startDate=2018-01-01&endDate=2018-02-31
  • Get orders on 1 January 2018
    • http://localhost/web/data/v2.1/order?date=2018-01-01

HTTP Status 200

POST /web/data/v2.1/order/duplicates

Check whether a list of order Ids already exists in a table

Required because users in the different UserGroups cannot view each other's orders. This checks whether an order with the same Id already exists in the table, regardless of UserGroup.

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Body

FieldRequiredTypeDefault ValueDescription
idsrequiredString[]List of order Ids to validate

Request Example

    // http://localhost/web/data/v2.1/order/duplicates
    {
        "ids": ["sg-12345","sg-does-not-exist"]
    }

Success Response

HTTP Status 200

FieldTypeDefault ValueDescription
String[]List of order Ids that already exist

Success Response Example

    // HTTP/1.1 200 OK
    ["sg-12345"]
