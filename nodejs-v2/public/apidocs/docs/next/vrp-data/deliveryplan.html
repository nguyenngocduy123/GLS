PUT /web/data/v2.1/plan

Approve a solution. An error will be returned if any of the jobs in the solution is already completed

This API assumes that the solution provided is correct and executable. Some cases are not validated, for example:

  • Jobs for a shipment can be assigned to different vehicles

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Body

FieldRequiredTypeDefault ValueDescription
solutionrequiredSolutionSolution generated by engine

Request Example

This is the bare minimum information required from the solution

    // http://localhost/web/data/v2.1/plan
    {
        "solution": {
            "routes": [{
                "vehicle_id": "Truck3Ft",
                "start_time": "2018-01-01T17:42+08:00",
                "end_time": "2018-01-01T20:33+08:00",
                "act": [{
                    "type": "pickup",
                    "job_id": "sg-12345"
                }]
            }],
            "unassigned_jobs": ["sg-4567"]
        }
    }

Success Response

HTTP Status 204

GET /web/data/v2.1/plan/problem

Converts data into a problem file that is based on engine input format

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Query String

FieldRequiredTypeDefault ValueDescription
daterequiredStringGet jobs on a date. Value must be in ISO8601 format
vehicleIdsoptionalStringComma-delimited string. Restrict vehicle utilisation to this list only

Request Example

  • Get problem file for a date
    • http://localhost/web/data/v2.1/plan/problem?date=2018-01-01
  • Get problem file that uses sg-12345 vehicle only
    • http://localhost/web/data/v2.1/plan/problem?date=2018-01-01&vehicleIds=sg-12345

Success Response

HTTP Status 200
