POST /web/data/v2.1/item

Create item(s). If any item fails to create, the entire list of items will not be created

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Body

FieldRequiredTypeDefault ValueDescription
recordrequiredObject[]List of items to insert into the database
record[].IdrequiredString(...30)Item identifier
record[].WeightrequiredFloatWeight of item
record[].DescriptionoptionalString(...1024)Description for item

Request Example

    // http://localhost/web/data/v2.1/item
    {
        "record": [{
            "Id": "Bag",
            "Weight": 5,
            "Description": "Black bag, volume 9L"
        }]
    }

Success Response

HTTP Status 200

FieldRequiredTypeDefault ValueDescription
requiredString[]List of Id of inserted records

Success Response Example

    // HTTP/1.1 200 OK
    ["Bag"] // one record inserted

PUT /web/data/v2.1/item/:itemId

Update item details. Item is identified based on itemId

Resource Info

Information
Response formatText
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Params

FieldRequiredTypeDefault ValueDescription
itemIdrequiredIntegerId column of Item table

Request Body

FieldRequiredTypeDefault ValueDescription
newValuesrequiredObjectValues to overwrite existing record with
newValues.WeightrequiredFloatWeight of item
newValues.DescriptionrequiredString(...1024)Description for item

Request Example

    // http://localhost/web/data/v2.1/item/Bag
    {
        "newValues": {
            "Weight": 3.4
        }
    }

Success Response

HTTP Status 200

FieldTypeDefault ValueDescription
IntegerNumber of updated items

Success Response Example

    // HTTP/1.1 200 OK
    1 // one record was updated

Information
Response formatJSON

Authorised RolePlanner, Driver
Rate limited?No
CSRF enabledYes

FieldRequiredTypeDefault ValueDescription

GET /web/data/v2.1/item

Get list of items based on where filter (optional)

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Query String

FieldRequiredTypeDefault ValueDescription
whereoptionalStringWhere conditions in JSON string

Request Example

  • Get all items
    • http://localhost/web/data/v2.1/item
  • Get items with weight 5
    • http://localhost/web/data/v2.1/order?where={"Weight":5}

Success Response

FieldRequiredTypeDefault ValueDescription
[]requiredItem[]List of items

Success Response Example

    // http://localhost/web/data/v2.1/item
    [{
        "Name": "Bag",
        "Weight": 5,
        "Description": "Black bag, volume 9L"
    }, {
        // ...
    }]
