POST /web/data/v2.1/vehicle

Create vehicle(s). If any vehicle fails to create, the entire list of vehicles will not be created

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Body

FieldRequiredTypeDefault ValueDescription
recordrequiredObject[]List of orders to insert into the database
record[].IdrequiredString(...50)Vehicle identifier. Cannot contain symbols other than dashes, underscores and spaces
record[].VehicleTypeIdrequiredIntegerType of vehicle based on VehicleType Id
record[].PlateNumberoptionalString(...10)Plate number of vehicle. Only letters, numbers and dashes are allowed
record[].StartAddressPostalrequiredString(6)Vehicle's start location
record[].StartAddressLatrequiredFloatLatitude of StartAddressPostal in degrees
record[].StartAddressLngrequiredFloatLongitude of StartAddressPostal in degrees
record[].EndAddressPostaloptionalString(6)nullVehicle's end location
record[].EndAddressLatoptionalFloatLatitude of EndAddressPostal in degrees. Required if EndAddressPostal is not null
record[].EndAddressLngoptionalFloatLongitude of EndAddressPostal in degrees. Required if EndAddressPostal is not null
record[].ReturnToEndAddressoptionalBooleanfalsetrue if vehicle needs to return to EndAddressPostal after completion of all jobs
record[].DriverUsernamerequiredString(...100)Username of driver assigned to this vehicle
record[].StartTimerequiredStringStart of vehicle operational time in HH:mm 24-hour format
record[].EndTimerequiredStringEnd of vehicle operational time in HH:mm 24-hour format
record[].MaxNumJobsrequiredInteger1000Maximum number of jobs that can be assigned to this vehicle
record[].UserGroupoptionalString(...100)Usergroup assigned with this vehicle

Request Example

    // http://localhost/web/data/v2.1/vehicle
    {
        "record": [{
            "Id": "Truck3Ft",
            "VehicleTypeId": 1,
            "PlateNumber": "SJJ1234M",
            "StartAddressPostal": "639588",
            "StartAddressLat": 1.3341,
            "StartAddressLng": 103.636,
            "EndAddressPostal": "639588",
            "StartAddressLat": 1.3341,
            "StartAddressLng": 103.636,
            "ReturnToEndAddress": true,
            "DriverUsername": "driver_doe",
            "StartTime": "09:00",
            "EndTime": "18:00",
            "MaxNumJobs": 30,
            "UserGroup": null
        }]
    }

Success Response

HTTP Status 200

FieldRequiredTypeDefault ValueDescription
requiredString[]List of Id of inserted records

Success Response Example

    // HTTP/1.1 200 OK
    ["Truck3Ft"] // one record inserted

POST /web/data/v2.1/vehicle/duplicates

Check whether a list of vehicle Ids already exists in a table

Required because users in the different UserGroups cannot view each other's vehicles. This checks whether an vehicle with the same Id already exists in the table, regardless of UserGroup.

Resource Info

Information
Response formatJSON
Requires authentication?Yes
Authorised RolePlanner
Rate limited?No
CSRF enabledYes

Request Body

FieldRequiredTypeDefault ValueDescription
idsrequiredString[]List of vehicle Ids to validate

Request Example

    // http://localhost/web/data/v2.1/vehicle/duplicates
    {
        "ids": ["Truck3Ft","Truck-does-not-exist"]
    }

Success Response

HTTP Status 200

FieldTypeDefault ValueDescription
String[]List of vehicle Ids that already exist

Success Response Example

    // HTTP/1.1 200 OK
    ["Truck3Ft"]
