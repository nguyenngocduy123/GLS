POST /web/problem/v2/optimize/:problemId

Do engine optimization on a problem

Resource Info

Information
Response formatsJSON
Requires authentication?Yes
Authorised RoleAdmin, Default, Planner
Rate limited?No
CSRF enabledYes

Query String

FieldRequiredTypeDefault ValueDescription
problemIdoptionalString_id of the problem

Request Body

FieldRequiredTypeDefault ValueDescription
enginerequiredStringchinhName of optimization solver to use. Accepted Values: chinh, siwei
solutionIdrequiredStrings_MMDDYYYY[_]HHmmUnique id of the solution to be returned
nThreadsrequiredString1Maximum number of threads to be used during optimization. Recommend: Based on number of cores on the machine
maxIterationsoptionalIntegerMaximum iterations allowed. The larger the number, the slower the generation, but increase in accuracy is not guaranteed
maxRunningTimeoptionalFloatUnit: Minutes. Maximum time the engine should run for. The engine will continue with the existing/final iteration before ending
costModeloptionalStringSpecify objective function cost model. Supported by chinh engine only
enableMinJobPenaltyoptionalBooleanfalseIndicates whether to pass minJobPenalty value to the engine
minJobPenaltyoptionalFloat0.3Use objective function with component for minimizing variation of number of jobs between vehicles. Supported by chinh engine only
sortByVehicleIdoptionalBooleanAssign jobs with a preference for vehicles with lower indexes. Supported by chinh engine only
constraintsoptionalStringList of constraints for engine. Concatenate strings using _
capacity - Ensure orders assigned to vehicles fit into vehicle
timeWindow - Ensure assigned vehicle arrive at order site within the order's time window of availability
priorityJob - Higher preference for higher priority jobs to be assigned to vehicle compared to lower priority jobs
restrictJobsToSpecificVehicles - If not specified, all vehicles are allowed to do any jobs. Allowed vehicles will be processed in the following sequence; allowed vehicle type, blocked vehicle type, allowed vehicles, blocked vehicles
consolidation - Group all jobs with the same postal code and process as if it's one order
maxNumJob - Limit number of jobs per vehicle
isSavedrequiredBooleantrueIndicates whether to save solution into MongoDb based on problem._id
problemrequiredProblemProblem details
dynamicSettingsoptionalObjectSettings for dynamic optimisation
dynamicSettings.moderequiredIntegerOnly required by siwei engine. Accepted Values: 0, 1, 2
0 - Evaluate solution without doing any optimisation
1 - Insert 1 service or shipment without changing original job sequence in the initial solution
2 - Insert 1 service or shipment, but the original job sequence may be modified
dynamicSettings.initialSolId*optionalString_id of the solution in problem.solutions to do dynamic optimisation on
dynamicSettings.initialSolIndex*optionalIntegerArray index of the solution in problem.solutions to do dynamic optimisation on
dynamicSettings.initialSol*optionalSolutionEntire, but simplified solution. Note: problem.solutions is not required if solution is specified here
selectedVehicleIdsoptionalString[]Only optimise with this list of vehicles and ignore the rest
selectedOrderIdsoptionalString[]Only optimise with this list of orders and ignore the rest

Success Response

HTTP Status 200

Note: Values based on distance and time are estimated based on the map data provided.

costs, time and distance values are for entire plan.

FieldRequiredTypeDefault ValueDescription
idrequiredStringSolution identifier
costsrequiredFloatUnit: Dollars. Total cost
fixed_costsrequiredFloatUnit: Dollars. Total fixed cost
variable_costsrequiredFloatUnit: Dollars. Total variable (dependent) cost
distancerequiredFloatUnit: Meters. Total distance covered
timerequiredFloatUnit: Seconds. Total time required
waiting_timerequiredFloatUnit: Seconds. Total waiting time
service_timerequiredFloatUnit: Seconds. Total service time
transport_timerequiredFloatUnit: Seconds. Total travel time
no_deliveriesrequiredIntegerTotal number of delivery jobs
no_pickupsrequiredIntegerTotal number of pickup jobs
no_unassigned_jobsrequiredIntegerTotal number of unassigned_jobs
unassigned_jobsrequiredString[]List of jobs id that are unable to be served
unassigned_reasonsrequiredObject[]List of jobs with the reasons on why job is unable to be served
unassigned_reasons[].job_idoptionalStringService or Shipment id
unassigned_reasons[].reasonoptionalStringReason for unassigned order
ignoredoptionalObjectList of properties that were ignored
ignored.vehiclesoptionalObjectDetails of ignored vehicles
ignored.vehicles.listoptionalString[]List of vehicles id that were ignored
ignored.vehicles.reasonsoptionalObject[]List of vehicles with reasons on why job is ignored (i.e. not passed to engine)
ignored.vehicles.reasons[].vehicle_idoptionalStringVehicle id
ignored.vehicles.reasons[].reasonoptionalStringReason for ignored vehicle
ignored.jobsoptionalObjectDetails of ignored orders
ignored.jobs.listoptionalString[]List of Service or Shipment id that were ignored
ignored.jobs.reasonsoptionalObject[]List of Service or Shipment id with reasons on why job is ignored (i.e. not passed to engine)
ignored.jobs.reasons[].vehicle_idoptionalStringService or Shipment id
ignored.jobs.reasons[].reasonoptionalStringReason for ignored order
calc_durationrequiredFloatUnit: Seconds. Time taken to generate a solution
calc_daterequiredStringDate that solution is generated on in ISO8601 format
no_routesrequiredIntegerTotal number of routes
routesrequiredRoute[]List of routes
